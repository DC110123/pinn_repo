# Test_B

Large-scale accuracy validation for the universal PINN model, generated by
`panel_accuracy_validation.py`.

## Contents

| File | Description |
| `B1.png` | Distribution of energy prediction error across 1,000 sampled panels |
| `panel_accuracy_validation.py` | Script used to run the batch accuracy check and generate the plot in this folder |

## What this validates

The goal of this test is to establish a rigorous accuracy distribution across
a large sample size (500+ panels), moving beyond single-panel spot checks to
**population-level statistics**. The benchmark compares the PINN's energy
yield predictions against high-fidelity physics solvers (`pvlib`) across the
CEC module database. By analyzing the Mean Absolute Error (MAE) and the error
distribution histogram, this test demonstrates that the model's accuracy is
consistent and reliable — not just accurate on a handful of hand-picked
panels, but robust enough for real-world industrial applications where
hardware varies.

**`B1.png` — Universal PINN Accuracy Distribution (1000 Panels)**

- X-axis: energy error (%), where **negative = the model underestimated**
  energy output and **positive = the model overestimated** it
- Y-axis: count of panels falling in each error bin
- Red dashed line: zero error (perfect prediction)
- Blue curve: kernel density estimate over the histogram

The distribution is sharply peaked at 0%, with the large majority of the
1,000 sampled panels falling within roughly ±1% error. A small number of
panels sit further out in the tails (out to roughly ±10%), representing
edge cases where the model performs worse — worth investigating if those
tail panels share common characteristics (e.g. unusual `R_sh_ref`, panel
technology, or module size).
